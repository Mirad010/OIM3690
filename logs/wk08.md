# Week 8 Log

This week I finished MP3 — Country Explorer.
Repo: https://github.com/Mirad010/country-explorer
Live: https://mirad010.github.io/country-explorer/

The app loads all countries, has search, region filter, sorting, a detail view
with live weather in the capital (second API — Open-Meteo), compare mode for
two countries, a flag quiz and favorites saved in localStorage.

AI collaboration note. AI generated my fetch code using the REST Countries API
(restcountries.com/v3.1/all). It worked in the examples, but when I deployed
the site to GitHub Pages, the page showed my error state: "Could not load
country data". At first I thought GitHub Pages was broken again like in MP2.
Then I opened the API URL directly in the browser and saw the real reason:
"This API version has been deprecated", and the new v5 version requires an API
key with only 500 free requests per month. A key cannot go into a public repo,
so instead we rewrote the code to use countries.dev, a free keyless API with
the same data. I also had to change the field names (name.common became name,
capital[0] became capital). The fix is in the commit "Switch to countries.dev
API after REST Countries deprecation".

This week I learned that a real API can change or die while you are using it,
and that the error state I built actually saved me — the page showed a clear
message instead of just breaking.
